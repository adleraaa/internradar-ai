# Archive

This folder is reserved for internship postings that have become **closed** or
**outdated** and have been moved out of the active dataset
(`../data/internships.json`).

## Status

**Nothing is archived yet.** The project is in early MVP setup and the active
dataset currently contains zero postings, so there is nothing to archive.

## Intended Workflow (future)

When the archival workflow is added, the process will be roughly:

1. A maintainer re-verifies a posting and finds it `Closed` or clearly outdated.
2. The entry is moved here (e.g. into a dated archive file) instead of being
   silently deleted, preserving a record of what was once listed.
3. The entry's `status`, `date_updated`, and `risk_flags` are updated to reflect
   that it is archived.

Archived entries are kept for transparency and history. They should **not** be
presented to students as active opportunities.
