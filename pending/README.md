# Pending Submissions

This folder holds **unreviewed** internship drafts in Markdown form, created
before they are reviewed and converted into JSON entries in
`../data/internships.json`.

## What goes here

- Draft Markdown files based on
  [`../templates/internship_submission_template.md`](../templates/internship_submission_template.md).
- Usually generated by [`../scripts/new_submission.py`](../scripts/new_submission.py),
  which writes a file named `YYYY-MM-DD_company_role.md` here.

## What this is NOT

- These drafts are **not** verified data. Nothing here has been added to the
  active dataset.
- A draft existing here does **not** mean the posting is real, open, or accurate.
  Every draft must still be manually reviewed against the official application
  page before it becomes a JSON entry.

## Typical flow

1. Generate a draft: `python ../scripts/new_submission.py` (run from `scripts/`).
2. Manually review the official application page and fill in the remaining
   fields, following the rules in the template.
3. Convert to JSON: `python scripts/submission_to_json.py pending/<file>.md`.
4. Append after validation: `python scripts/append_entry.py <json-file>`.
5. Optionally move the reviewed Markdown to `../reviewed/`.

Drafts that are abandoned or rejected can simply be deleted from this folder.
